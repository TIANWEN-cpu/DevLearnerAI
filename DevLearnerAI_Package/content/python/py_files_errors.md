# 文件、异常与调试心法

## 学习目标
- 会安全地读写文本文件
- 理解为什么异常处理是工程能力的一部分
- 知道报错时如何定位问题

## 读写文件
最常用的方式是 `with open(...)`：

```python
with open("notes.txt", "r", encoding="utf-8") as file:
    content = file.read()
```

写文件类似：

```python
with open("notes.txt", "w", encoding="utf-8") as file:
    file.write("hello")
```

`with` 的价值在于：即使中间出错，文件也能被正确关闭。

## 为什么会报错
真实程序永远不只处理理想输入。以下情况都可能失败：
- 文件不存在
- 用户输入不是数字
- 除数为 0
- 外部接口超时

## try / except
```python
try:
    value = 10 / 0
except ZeroDivisionError:
    print("不能除以 0")
```

异常处理不是为了“掩盖错误”，而是为了：
- 给用户更友好的反馈
- 在合理的位置兜底
- 保留继续运行的机会

## 调试心法
定位 bug 时建议按顺序做：

1. 先重现问题
2. 缩小出错范围
3. 打印关键变量
4. 验证你的假设
5. 修完后再跑一遍

## 一个常见坏习惯
不要一上来就写：

```python
except:
    pass
```

这样等于把真正的问题藏起来，后续更难查。

## 本课总结
当你开始处理文件和错误时，说明你已经在接触“真实世界输入”。这一步会让你的代码从玩具脚本变成可用工具。
