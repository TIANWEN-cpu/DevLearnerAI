# Python 学习地图与环境准备

## 学习目标
- 搞清楚解释器、脚本、包、第三方库分别是什么
- 知道为什么先学语言基础，再学数据库和项目
- 能在终端运行一个 `.py` 文件

## 先建立正确心智模型
很多初学者一上来就想学爬虫、Web、AI，但真正卡住你的通常不是“高级框架”，而是最基础的三件事：

1. 数据长什么样
2. 代码按什么顺序执行
3. 报错时怎么定位

如果这三件事没稳住，后面的数据库、接口、框架都会变成“照着敲也不会改”的状态。

## Python 到底是什么
- Python 是一种解释型语言：代码不是先编译成 exe 再运行，而是交给解释器逐句执行
- 解释器就是你安装的 Python 程序本身
- `.py` 文件是脚本，是你写的源代码
- `pip` 是安装第三方库的工具，不是 Python 语言本身

## 最小工作流
```python
print("hello, python")
```

把它保存成 `hello.py` 后，在终端运行：

```bash
python hello.py
```

如果屏幕上出现输出，你就完成了从“安装环境”到“代码被解释器执行”的第一条链路。

## 推荐学习顺序
1. 变量、类型、表达式
2. 条件和循环
3. 函数与模块
4. 文件、异常、调试
5. 类与对象
6. 测试、环境、项目结构
7. 数据库
8. 融合项目

## 常见误区
- 误区一：先追热点再补基础
  结果是能复制教程，但一改就坏
- 误区二：把库当语言学
  会用某个框架，不等于会编程
- 误区三：只看不写
  编程必须靠手感积累

## 练习建议
- 自己新建一个 `hello.py`
- 再写一个 `about_me.py`，打印你的名字和学习目标
- 试着故意写错一行，观察报错长什么样

## 本课总结
这节课不是在教你“炫酷技术”，而是在帮你搭第一层地基。后面每一课都要围绕一个问题展开：我是否真的理解代码为什么这样运行？
