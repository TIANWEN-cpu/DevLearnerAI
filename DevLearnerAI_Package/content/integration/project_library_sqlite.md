# 项目二：图书管理系统（Python + SQLite）

## 项目目标
实现一个本地图书管理系统，支持新增图书、查询图书、修改状态、删除图书，并把数据存到 SQLite。

## 为什么它是关键过渡项目
这是你第一次把两条主线真正合并：
- Python 负责业务逻辑、输入输出和结构组织
- 数据库负责持久化和查询

## 推荐表结构
```sql
CREATE TABLE books (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    author TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'available'
);
```

## 推荐代码分层
- `db.py`：数据库连接和 SQL 执行
- `service.py`：图书业务逻辑
- `main.py`：程序入口

哪怕项目很小，也要开始练习“别把所有代码都塞进一个文件”。

## 第一版功能
1. 新增图书
2. 列出所有图书
3. 按 id 修改图书状态
4. 删除图书

## 项目验收标准
- 表结构能正常创建
- 所有新增图书能持久化
- 状态修改后重新打开程序仍正确
- 删除功能不会误删多条记录

## 常见坑
- SQL 直接字符串拼接
- 忘记 `commit()`
- 没有主键就尝试修改某条记录
- 业务逻辑和数据库语句完全耦合在一起

## 完成后你应该复盘什么
- 你的函数是否职责单一
- 数据库操作是否都参数化
- 程序入口是否清晰
- 如果明天要加“借阅记录”功能，结构还能不能扩展

## 学习收获
做完这个项目，你会开始真正拥有“我已经能做一个小型业务系统”的感觉。
