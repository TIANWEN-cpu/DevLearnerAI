# 项目一：命令行任务管理器

## 项目目标
做一个命令行 Todo 工具，支持新增任务、查看任务、完成任务、删除任务，并把数据保存到本地文件。

## 这个项目为什么重要
它是最适合巩固 Python 基础的项目之一，因为它会同时用到：
- 函数拆分
- 列表与字典
- 文件读写
- 异常处理
- 程序入口组织

## 推荐 MVP 范围
第一版只做四个命令：
- `add`
- `list`
- `done`
- `delete`

不要一开始就加优先级、标签、提醒时间。先把最小闭环做出来。

## 数据结构建议
可以先用列表存字典：

```python
[
    {"title": "学完函数", "done": false}
]
```

然后保存为 JSON 文件。

## 拆分建议
- `load_tasks()`
- `save_tasks(tasks)`
- `add_task(tasks, title)`
- `list_tasks(tasks)`
- `mark_done(tasks, index)`

## 验收标准
- 能新增任务
- 能看到任务列表
- 能修改完成状态
- 关闭程序后数据还在

## 进阶方向
- 支持命令行参数
- 支持按状态筛选
- 增加日志和测试

## 学习收获
做完这个项目，你应该明显感觉到：函数和文件处理终于不是“知识点”，而是能撑起一个完整工具的真实能力。
