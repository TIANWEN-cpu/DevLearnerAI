# 项目三：FastAPI + 数据库学习记录后端

## 项目目标
做一个学习记录后端 API，支持课程列表、学习进度、练习记录三类资源。

## 为什么要做这个项目
因为从这里开始，你会真正接近后端开发的日常：
- 接口设计
- 请求与响应
- 数据库访问
- 分层组织
- 测试与可维护性

## 建议的资源设计
- `GET /lessons`
- `GET /progress`
- `POST /progress`
- `GET /attempts`
- `POST /attempts`

## 建议的目录结构
```text
app/
  main.py
  routers/
  services/
  repositories/
  schemas/
  models/
```

## 第一版只做最小可用
不要一开始就做登录、权限、复杂部署。先把下面三件事做好：

1. 能返回课程列表
2. 能记录学习完成状态
3. 能记录练习得分

## 验收标准
- 接口命名清晰
- 至少有一个 POST 和一个 GET 闭环
- 数据能真实写入数据库
- 代码按职责分层，不把所有逻辑写在路由函数里

## 进阶方向
- 加入分页
- 加入输入校验
- 加入单元测试
- 把 SQLite 换成 PostgreSQL

## 学习收获
完成这个项目后，你就不再只是“会写几个脚本”，而是开始具备后端项目的结构感和交付感。
