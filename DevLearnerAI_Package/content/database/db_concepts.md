# 表、行、列与主键思维

## 学习目标
- 理解数据库的基本组成
- 知道主键和唯一标识的重要性
- 开始形成“先建模，再写 SQL”的习惯

## 数据库在解决什么问题
程序运行时的数据在内存里，程序一关就没了。数据库解决的是：

- 数据持久化
- 数据结构化存储
- 查询和统计
- 多条数据之间的关系管理

## 表、行、列
- 表：一类数据的集合
- 行：一条记录
- 列：某个字段

比如学生表：

| id | name | score |
|----|------|-------|
| 1  | Tom  | 88    |

## 主键为什么重要
主键的作用是唯一标识一条记录。没有主键，你会很难稳定地更新、删除或关联数据。

常见做法：
- 整数自增 id
- 有业务意义的唯一编码

## 一个建模问题
如果你要设计“图书借阅系统”，至少会涉及：
- 图书表
- 用户表
- 借阅记录表

这就是关系建模的起点。

## 初学者常见误区
- 把多个实体混进一张表
- 没有唯一标识
- 还没想清楚字段含义就开始写 SQL

## 本课总结
数据库学习不能只背命令。先把数据组织方式想明白，后面的查询、连接、索引才有意义。
