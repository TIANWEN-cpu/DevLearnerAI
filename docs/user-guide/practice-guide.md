## 练习系统概览

DevLearner AI 的练习系统提供 **176+ 道练习题**，覆盖全部 8 条学习路线，支持多种编程语言的自动评测。

### 支持的语言和评测方式

| 语言 | 评测方式 | 说明 |
|------|----------|------|
| Python | 沙箱执行 + 测试用例 | 在安全沙箱中实际运行代码，验证输出结果 |
| SQL | 真实数据库比对 | 在内存 SQLite 数据库中执行查询，比对结果集 |
| C / C++ | 结构检查 | 检查代码关键字和语法结构 |
| C# | 结构检查 | 检查代码关键字和语法结构 |

### 难度分级

| 难度 | 说明 | 适合人群 |
|------|------|----------|
| 基础 | 基本语法和概念练习 | 初学者 |
| 进阶 | 综合运用多个知识点 | 有一定基础的学习者 |
| 综合 | 复杂问题和工程实践 | 进阶学习者 |

## 评测机制详解

### Python 代码评测

Python 代码在安全沙箱中执行，评测流程如下：

1. **AST 预检** -- 使用 Python AST 模块解析代码，拦截危险操作：
   - 禁止 `os.system`、`subprocess`、`eval`、`exec` 等危险函数
   - 禁止访问 `__class__`、`__bases__`、`__subclasses__` 等双下划线属性
   - 限制可导入模块白名单（`math`、`json`、`re` 等标准库）
2. **沙箱执行** -- 在受限环境中执行代码
   - 文件操作限制在临时工作目录内
   - 标准输出限制为 12KB
   - 执行超时为 3 秒
3. **结果验证** -- 比对代码输出与预期结果

### SQL 代码评测

SQL 代码在内存 SQLite 数据库中执行：

1. **语法解析** -- 检查 SQL 语句语法
2. **数据准备** -- 根据练习配置创建测试数据
3. **执行查询** -- 在内存数据库中执行用户的 SQL 语句
4. **结果比对** -- 将查询结果与预期结果集进行标准化比对
5. **副作用验证** -- 对于 DDL 操作（CREATE、ALTER 等），验证表结构变化

### C / C# 代码评测

由于运行环境限制，C 和 C# 代码采用结构检查方式评测：

1. **关键字检查** -- 验证是否使用了要求的关键字
2. **结构检查** -- 检查代码是否包含必要的语法结构
3. **模式匹配** -- 验证代码是否符合题目要求的模式

---

## 评分标准

评测结果包含多维度评分：

| 评分维度 | 权重 | 说明 |
|----------|------|------|
| 语法检查 | 基础 | 代码是否能正确解析 |
| 结构检查 | 基础 | 代码是否包含必要的结构 |
| 运行结果 | 核心 | 代码执行结果是否正确 |
| 测试用例 | 核心 | 测试用例通过率 |

### 评测反馈

每次评测会返回详细的反馈信息：

- **通过/未通过** -- 整体评测结果
- **得分** -- 0-100 分制的评测分数
- **反馈信息** -- 详细的评测反馈，包括：
  - 语法错误提示
  - 运行时错误信息
  - 预期输出与实际输出的差异
  - 改进建议

### 评测超时

- Python 代码执行超时：3 秒
- 评测流程超时：4 秒
- 超时后系统会提示代码可能存在死循环或大量计算

## 草稿自动保存

### 自动保存机制

练习系统提供草稿自动保存功能：

- 每次编辑代码后，系统自动保存当前草稿
- 关闭练习页面或退出应用后，草稿会持久化存储
- 重新打开同一练习时，自动恢复上次的草稿

### 草稿存储

- 草稿存储在数据库的 `exercise_drafts` 表中
- 每个练习独立保存草稿
- 提交评测不影响草稿内容

### 使用建议

- 不用担心代码丢失，放心编写
- 可以分多次完成一个练习
- 提交评测后如不满意，可以继续编辑草稿

## 练习技巧

### 1. 先学习再练习

每个练习都关联了相关的课程内容。建议先完成对应课程的学习，再进行练习。

### 2. 善用提示

练习题目中包含提示信息。如果遇到困难，先查看提示再尝试解答。

### 3. 多次迭代

不要期望一次通过。根据评测反馈修改代码，逐步改进：

1. 提交代码
2. 查看评测反馈
3. 根据反馈修改代码
4. 再次提交

### 4. 利用 AI 导师

遇到困难时，可以使用 AI 导师功能：
- 点击"分析当前代码"，让 AI 诊断问题
- 直接向 AI 导师提问
- 请求解题思路和算法建议

### 5. 按难度递进

建议按难度等级递进练习：
1. 先完成所有基础练习
2. 再挑战进阶练习
3. 最后攻克综合练习

### 6. 定期复习

利用仪表盘查看练习平均分，定期回顾之前做过的练习，巩固所学知识。
