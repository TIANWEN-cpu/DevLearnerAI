## AI 导师简介

AI 导师是 DevLearner AI 的核心功能之一，基于 OpenAI 兼容 API 提供智能编程辅导。它可以：

- **解答编程问题** -- 解释概念、分析代码、提供示例
- **辅助课程学习** -- 详细讲解课程内容，补充扩展知识
- **帮助调试代码** -- 分析错误信息，提供修复建议
- **提供学习建议** -- 根据你的学习进度推荐下一步
- **拆解项目任务** -- 帮助规划项目结构和实施步骤

---

## 使用模式

AI 导师提供两种使用模式，你可以根据需要切换：

### 模式一：AI 工作台（独立页面）

- 全屏独立页面，提供更大的对话区域
- 适合深度学习和长时间对话
- 完整的功能体验

### 模式二：侧边助手（Dock 面板）

- 停靠在界面右侧的面板
- 可以一边查看课程/练习，一边与 AI 对话
- 适合边学边问的场景

### 切换模式

在 AI 导师界面中，可以通过切换按钮在两种模式之间自由切换。

## 配置 AI 服务

### 首次配置

1. 点击 AI 导师界面中的 **AI 设置** 按钮
2. 填写以下信息：

| 配置项 | 说明 | 必填 |
|--------|------|------|
| API Host | API 服务地址，必须以 `https://` 开头 | 是 |
| API Key | API 访问密钥 | 是 |
| 模型 | 选择使用的 AI 模型 | 是 |

3. 点击 **测试连接** 确认配置正确
4. 点击 **获取模型** 加载可用模型列表
5. 选择模型后即可开始使用

### 安全说明

- API 密钥通过 Windows Credential Manager 或 keyring 安全存储
- 所有 API 通信强制使用 HTTPS
- 启用 TLS 证书验证
- 连接状态有 5 分钟缓存，减少重复测试

## 开始对话

### 发送消息

1. 在对话输入框中输入你的问题
2. 点击 **发送** 按钮或按 Enter 键
3. AI 将生成回复并显示在对话区域

### 对话界面功能

| 功能 | 说明 |
|------|------|
| Markdown 渲染 | AI 回复中的标题、列表、表格等格式正确显示 |
| 代码语法高亮 | 代码块自动高亮显示 |
| 滚动浏览 | 支持上下滚动查看对话历史 |

### 对话上下文

AI 导师具有上下文感知能力：
- 自动加载最近 12 条历史消息作为对话上下文
- 可以引用之前对话中的内容
- 支持多轮连续对话

## 会话管理

### 多会话支持

你可以按主题创建多个独立的对话会话，例如：

- "Python 调试" -- 专门讨论 Python 调试技巧
- "数据库设计" -- 讨论数据库相关问题
- "项目拆解" -- 规划和分解项目任务
- "算法分析" -- 讨论算法和数据结构

### 创建新会话

在 AI 导师界面中点击 **新建会话** 按钮，输入会话主题即可创建。

### 切换会话

在会话列表中点击不同的会话标题即可切换。每个会话独立保存对话历史。

### 会话持久化

所有会话数据存储在数据库中：
- `mentor_sessions` 表保存会话信息
- `mentor_messages` 表保存对话消息
- 重新打开应用后，历史会话自动恢复

## 快捷提问功能

AI 导师内置了快捷提问功能，帮助你快速获取帮助：

### 解释当前课程

自动获取当前正在阅读的课程内容，让 AI 进行详细讲解：
- 概念解释
- 代码逐行分析
- 补充示例
- 常见误区

### 分析当前代码

自动获取当前练习中的代码，让 AI 进行分析：
- 代码逻辑解读
- 潜在问题诊断
- 性能优化建议
- 代码风格改进

### 拆解当前项目

针对当前项目，让 AI 帮助规划：
- 任务分解
- 模块设计
- 实施步骤
- 技术选型建议

### 使用方法

1. 在课程阅读器或练习页面中打开 AI 导师
2. 点击对应的快捷提问按钮
3. AI 会自动获取上下文并生成回答

## 知识库系统

AI 导师具备知识库功能，增强回答的准确性和针对性：

### 知识库类型

| 类型 | 说明 |
|------|------|
| 基础知识库 | 项目内置的基础知识和常见问题 |
| 个性知识库 | 你的个人学习记录和笔记 |
| 扩展文件知识库 | 可自定义添加的知识文件 |

### 上下文构建

发送消息时，系统会自动构建上下文：
1. 加载当前课程摘要
2. 加载学习进度信息
3. 加载相关知识库文件
4. 加载最近 12 条对话历史
5. 组装为完整的系统上下文发送给 API

### 知识库文件管理

在 AI 设置中可以管理知识库文件：
- 添加自定义知识文件
- 删除不需要的文件
- 文件存储在 `mentor_knowledge_files` 表中

## 最佳实践

### 1. 明确提问

好的提问示例：
- "请解释 Python 中列表推导式的语法和使用场景"
- "这段代码为什么报错 `IndexError`？"
- "如何用 SQL 实现分组统计？"

避免模糊的提问：
- "帮我看一下代码"
- "这道题怎么做？"

### 2. 提供上下文

如果问题涉及具体代码，请：
- 复制相关的代码片段
- 描述你的预期结果
- 说明实际运行结果

### 3. 分步提问

复杂问题建议分步提问：
1. 先请求概念解释
2. 再请求代码示例
3. 最后请求优化建议

### 4. 利用会话管理

为不同主题创建独立会话，保持每个会话的专注度。

### 5. 结合课程学习

- 学习新课程前，让 AI 预览课程要点
- 学习过程中，随时提问不理解的部分
- 学完后，让 AI 出几道练习题巩固

### 6. 代码审查

完成练习后，可以让 AI 审查你的代码：
- 代码风格是否规范
- 是否有更好的实现方式
- 是否存在潜在的 bug

## 常见问题

### Q: AI 回复很慢怎么办？

可能原因：
- 网络延迟 -- 检查网络连接
- API 服务器负载 -- 尝试切换模型或稍后重试
- 问题过于复杂 -- 尝试将问题拆分为更小的子问题

### Q: AI 回复不准确怎么办？

- 尝试重新组织提问方式
- 提供更多上下文信息
- 使用知识库功能增强 AI 的领域知识
- 让 AI "一步一步思考"

### Q: 对话历史会丢失吗？

不会。所有对话历史存储在本地数据库中，重新打开应用后自动恢复。

### Q: 如何清除对话历史？

在会话列表中右键点击会话，选择删除即可清除该会话的所有历史消息。

### Q: 支持哪些 AI 模型？

支持所有 OpenAI 兼容 API 提供的模型，包括 GPT-4o、Claude（通过兼容网关）等。具体可用模型取决于你的 API 服务商。
