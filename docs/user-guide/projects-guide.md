## 项目系统概览

融合项目是 DevLearner AI 的重要特色功能，旨在帮助你从单点知识走向可交付的真实项目，培养工程实践能力。

### 项目设计理念

- **真实场景** -- 每个项目模拟真实开发场景
- **由浅入深** -- 从简单的 CLI 工具到复杂的 Web 应用
- **知识融合** -- 将多个技术栈的知识串联运用
- **工程导向** -- 注重代码质量、项目结构和开发流程

### 项目数量

当前提供 **10 个真实项目**，覆盖以下技术方向：
- 命令行工具开发
- 数据采集与处理
- Web API 开发
- 桌面应用开发
- AI 应用集成

### 推荐学习路径

**初学者路径**：
1. P1 Todo CLI -- 入门项目，练习基本 Python 编程
2. P2 网络爬虫 -- 学习 HTTP 请求和数据解析
3. P4 数据分析 -- 掌握数据处理基础

**进阶路径**：
1. P3 ORM 数据库 -- 理解数据库交互
2. P5 FastAPI 服务 -- 学习 API 开发
3. P8 音乐播放器 -- 练习 GUI 开发

**高级路径**：
1. P6 Celery 任务队列 -- 异步编程
2. P7 Django 应用 -- 全栈开发
3. P10 AI 对话助理 -- AI 集成

## 如何使用项目

### 第 1 步：进入项目页面

在左侧导航栏中点击 **融合项目**，浏览可用项目列表。

### 第 2 步：选择项目

根据你的学习进度和兴趣选择合适的项目。注意查看项目的难度等级和前置要求。

### 第 3 步：阅读项目文档

每个项目都配有详细的 Markdown 文档，包含：
- 项目简介和目标
- 功能需求
- 技术要点
- 实施步骤
- 扩展挑战

### 第 4 步：动手实践

按照项目文档的指引，逐步完成项目开发：
1. 搭建项目结构
2. 实现核心功能
3. 测试和调试
4. 优化和扩展

### 第 5 步：使用 AI 辅助

在项目开发过程中，可以随时使用 AI 导师：
- 点击"拆解当前项目"获取任务分解
- 咨询技术选型建议
- 请求代码审查
- 讨论架构设计

---

## 项目文档结构

每个项目的文档遵循统一的结构：

```markdown
# 项目名称

## 项目简介
概述项目的目标和价值

## 功能需求
列出需要实现的功能点

## 技术要点
讲解涉及的核心技术

## 实施步骤
分步骤指导项目开发

## 扩展挑战
可选的进阶功能
```

### 文档格式

- 项目文档以 Markdown 格式组织
- 存储在 `content/projects/` 目录下
- 支持代码语法高亮渲染

## 结合 AI 导师使用

AI 导师是完成项目的重要助手：

### 项目规划阶段

使用"拆解当前项目"功能，获取：
- 任务分解清单
- 模块划分建议
- 开发顺序推荐
- 时间估计

### 开发过程中

随时向 AI 导师提问：
- "如何实现 XXX 功能？"
- "这个错误怎么解决？"
- "这段代码可以怎么优化？"
- "这个模块的测试应该怎么写？"

### 完成后

让 AI 导师审查你的代码：
- 代码质量评估
- 性能优化建议
- 安全性检查
- 最佳实践推荐

## 学习建议

### 1. 不要只看不做

项目文档是指导，不是答案。务必亲自动手编写代码。

### 2. 先完成核心功能

优先实现项目的核心功能，再考虑扩展功能。不要一开始就追求完美。

### 3. 写下你的思路

在开始编码前，先在纸上或笔记中规划你的实现方案。

### 4. 遇到困难时

1. 先回顾相关的课程内容
2. 尝试自己搜索解决方案
3. 向 AI 导师求助
4. 分析 AI 的建议，理解原理后再应用

### 5. 完成后回顾

每个项目完成后，花时间回顾：
- 学到了哪些新知识
- 遇到了哪些问题，如何解决
- 有哪些可以改进的地方
- 下一个项目需要补强哪些知识

### 6. 多做扩展挑战

完成基本功能后，尝试文档中的扩展挑战，进一步提升技能。
