## 应用启动问题

### 问题：启动时报 "No module named 'PyQt5'"

**原因**：PyQt5 未安装或虚拟环境未激活。

**解决方案**：
```bash
pip install PyQt5>=5.15
```

如果使用虚拟环境，确认已激活：
```bash
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # Linux / macOS
```

### 问题：启动后立即闪退

**原因**：可能是依赖缺失、数据库损坏或配置错误。

**解决方案**：
1. 使用开发模式启动查看详细日志：
   ```bash
   python dev_main.py
   ```
2. 检查控制台输出的错误信息
3. 确认所有依赖已安装：
   ```bash
   pip install -r requirements.txt
   ```

### 问题：启动时报 "Permission denied"

**原因**：数据目录没有写入权限。

**解决方案**：
1. 确认 `%APPDATA%/DevLearnerAI/` 目录有写入权限
2. 以管理员身份运行（不推荐长期使用）
3. 检查杀毒软件是否阻止了文件写入

### 问题：Python 版本不兼容

**原因**：Python 版本低于 3.9。

**解决方案**：
```bash
python --version
```
如果版本低于 3.9，请升级 Python。推荐使用 3.12 版本。

## API 连接问题

### 问题：AI 助手显示 "连接失败"

**原因**：API 配置不正确或网络问题。

**排查步骤**：

1. **检查 API Host 格式**
   - 必须以 `https://` 开头
   - 不支持 HTTP 连接（出于安全考虑）
   - 示例：`https://api.openai.com`

2. **检查 API Key**
   - 确认 Key 没有多余的空格
   - 确认 Key 仍然有效（未过期或被撤销）
   - 可在 AI 设置中重新输入

3. **使用测试连接功能**
   - 点击 AI 设置中的"测试连接"按钮
   - 查看返回的详细错误信息

4. **检查网络连接**
   - 确认可以访问互联网
   - 检查防火墙是否阻止了 HTTPS 连接
   - 如果使用代理，确认代理配置正确

### 问题：测试连接超时

**原因**：网络延迟或 API 服务器响应慢。

**解决方案**：
- 检查网络连接质量
- 尝试更换 API 端点
- 稍后重试（连接测试结果缓存 5 分钟）

### 问题：获取模型列表失败

**原因**：API 端点不支持 `/models` 接口，或权限不足。

**解决方案**：
- 确认 API 服务支持 OpenAI 兼容的模型列表接口
- 确认 API Key 有权限访问模型列表
- 手动输入模型名称

### 问题：SSL 证书错误

**原因**：系统证书链不完整或 API 服务器证书有问题。

**解决方案**：
- 更新系统证书
- 检查系统时间是否正确
- 如果是自签名证书，需要额外配置

## 代码执行沙箱问题

### 问题：代码执行超时

**现象**：提交代码后提示"执行超时"。

**原因**：代码执行时间超过 3 秒限制（评测超时 4 秒）。

**解决方案**：
- 检查代码中是否有死循环
- 优化算法，减少计算量
- 检查是否有大量 I/O 操作

### 问题：代码中不允许使用 import

**现象**：代码中的 `import` 语句被拦截。

**原因**：沙箱限制了可导入的模块白名单。

**允许的模块**：
- `math` -- 数学运算
- `json` -- JSON 处理
- `re` -- 正则表达式
- 其他标准库模块（根据白名单配置）

**解决方案**：
- 使用沙箱允许的标准库模块
- 不使用第三方库
- 使用内置函数完成需求

### 问题：不允许访问 __class__ 等属性

**现象**：代码中的 `__class__`、`__bases__` 访问被拦截。

**原因**：沙箱禁止访问双下划线属性，防止沙箱逃逸。

**解决方案**：避免使用双下划线属性，使用正常的编程方式。

### 问题：文件操作被拒绝

**现象**：代码中的文件读写操作失败。

**原因**：文件操作限制在临时工作目录内。

**解决方案**：
- 沙箱环境中的文件操作仅限于临时目录
- 练习通常不需要文件操作
- 使用标准输入输出代替文件操作

### 问题：输出被截断

**现象**：代码输出不完整。

**原因**：标准输出限制为 12KB。

**解决方案**：
- 减少输出内容
- 只输出必要的结果
- 避免大量调试输出

## 课程内容加载问题

### 问题：课程列表为空

**原因**：`content/` 目录或 `course_map.json` 文件缺失。

**解决方案**：
1. 确认 `content/` 目录存在
2. 确认 `content/metadata/course_map.json` 文件存在且格式正确
3. 如果使用可执行文件，确认 `content/` 目录已正确打包

### 问题：课程内容显示空白

**原因**：Markdown 文件缺失或格式错误。

**解决方案**：
1. 检查课程对应的 `.md` 文件是否存在
2. 确认文件编码为 UTF-8
3. 检查 Markdown 语法是否有严重错误

### 问题：代码高亮不正确

**原因**：代码块缺少语言标识或使用了不支持的语言。

**解决方案**：
- 在代码块的开头指定语言标识：
  ````
  ```python
  code here
  ```
  ````
- 使用支持的语言标识（python、c、cpp、csharp、sql 等）

### 问题：课程显示乱码

**原因**：文件编码不是 UTF-8。

**解决方案**：
- 将课程文件转换为 UTF-8 编码
- 检查文件是否有 BOM 头

## 练习评测问题

### 问题：SQL 练习评测结果不正确

**原因**：SQL 语句与预期结果不匹配。

**排查步骤**：
1. 检查 SQL 语法是否正确
2. 比对查询结果与预期结果的数据类型
3. 注意大小写敏感性
4. 检查排序和去重

### 问题：C/C# 练习评测不通过

**原因**：代码缺少必要的关键字或结构。

**解决方案**：
- 查看题目要求中列出的必要关键字
- 确保代码包含所有要求的关键字
- 检查代码结构是否符合要求

### 问题：草稿丢失

**原因**：数据库异常或草稿表损坏。

**解决方案**：
1. 检查数据库文件完整性
2. 草稿保存在 `exercise_drafts` 表中
3. 如数据库损坏，可删除 `app.db` 重新开始（注意会丢失所有数据）

## 数据库问题

### 问题：数据库文件路径

数据库文件位置：
- **主数据库**：`%APPDATA%/DevLearnerAI/data/app.db`
- **旧版数据库**：`db/learner.db`（会自动迁移）

其中 `%APPDATA%` 通常为：
- `C:\Users\<用户名>\AppData\Roaming`

### 问题：数据库损坏

**现象**：应用报数据库相关错误。

**解决方案**：
1. 关闭应用
2. 备份 `%APPDATA%/DevLearnerAI/data/app.db`
3. 删除 `app.db`
4. 重新启动应用（会创建新数据库）

**注意**：删除数据库会导致所有学习进度、练习记录和 AI 对话历史丢失。

### 问题：旧版数据库未迁移

**现象**：升级后学习进度丢失。

**解决方案**：
1. 检查旧版数据库 `db/learner.db` 是否存在
2. 应用启动时会自动检测并迁移旧数据库
3. 如果自动迁移失败，可手动复制 `db/learner.db` 到 `%APPDATA%/DevLearnerAI/data/app.db`

### 问题：并发写入冲突

**现象**：多线程操作时报数据库锁定错误。

**说明**：应用使用 WAL 模式和写操作锁来避免此类问题。如果仍然遇到，请重启应用。

## 界面显示问题

### 问题：界面元素错乱或重叠

**原因**：DPI 缩放或样式问题。

**解决方案**：
1. 尝试调整系统 DPI 设置
2. 重启应用
3. 如果使用可执行文件，尝试设置兼容性选项

### 问题：代码块显示不正确

**原因**：Markdown 渲染或语法高亮问题。

**解决方案**：
1. 检查课程文件中的代码块格式是否正确
2. 确认代码块使用了正确的围栏语法（三个反引号）
3. 尝试重新启动应用

### 问题：侧边栏导航不响应

**原因**：界面线程阻塞或资源加载问题。

**解决方案**：
1. 等待资源加载完成
2. 重启应用
3. 检查日志中是否有错误信息

## 打包和分发问题

### 问题：PyInstaller 打包失败

**常见原因和解决方案**：

1. **缺少 hidden imports**
   - 在 `.spec` 文件中添加缺失的模块

2. **数据文件未包含**
   - 检查 `.spec` 文件中的 `datas` 配置

3. **路径问题**
   - 使用绝对路径
   - 确认所有引用的文件存在

### 问题：可执行文件运行时报错

**排查步骤**：
1. 使用命令行运行可执行文件，查看输出
2. 检查 `--console` 模式的输出
3. 查看 `%APPDATA%/DevLearnerAI/logs/` 中的日志

## 日志查看

### 日志位置

日志文件保存在：`%APPDATA%/DevLearnerAI/logs/`

### 开启 DEBUG 日志

使用开发模式启动可以查看更详细的日志：

```bash
python dev_main.py
```

### 日志级别

| 级别 | 说明 |
|------|------|
| DEBUG | 详细信息，用于调试 |
| INFO | 一般信息 |
| WARNING | 警告信息 |
| ERROR | 错误信息 |

生产模式（`main.py`）默认使用 INFO 级别，开发模式（`dev_main.py`）使用 DEBUG 级别。

### 常见日志信息

- **数据库操作日志** -- 记录数据库读写操作
- **API 通信日志** -- 记录 AI API 请求和响应
- **课程加载日志** -- 记录课程内容加载过程
- **评测流程日志** -- 记录练习评测步骤
