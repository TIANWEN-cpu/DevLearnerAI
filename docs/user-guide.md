## 快速开始

### 第一步：安装与启动

1. 确保已安装 Python 3.9 或更高版本
2. 安装依赖：`pip install -r requirements.txt`
3. 启动应用：`python main.py`

### 第二步：浏览课程

1. 启动后默认进入**首页**（学习仪表盘）
2. 点击左侧导航栏的**学习路径**进入课程体系
3. 选择感兴趣的技术栈（如 Python、数据库等）
4. 展开模块，点击课程开始学习

### 第三步：做练习

1. 点击左侧导航栏的**练习中心**
2. 选择技术栈和难度筛选练习
3. 阅读题目描述，在代码编辑器中编写代码
4. 点击**运行**查看输出，点击**提交**进行评测
5. 查看评测反馈，了解得分和改进建议

### 第四步：配置 AI 助手

1. 点击顶部**AI 工作台**或使用快捷键 `Ctrl+Shift+A`
2. 点击**AI 设置**按钮
3. 在**模型连接**标签页中填写：
   - API Host（必须以 `https://` 开头）
   - API Key
4. 点击**获取模型**并选择要使用的模型
5. 点击**保存配置**

## 功能详解

### 学习仪表盘

学习仪表盘是应用的首页，提供学习进度的全局概览。

**功能包括**:

- **欢迎区域**: 显示当前日期和学习状态
- **统计卡片**: 已完成课程数、练习平均分、连续学习天数
- **技术栈入口**: 快速进入各技术栈的学习路径
- **快捷操作**: 快速跳转到常用功能

点击技术栈卡片可直接跳转到对应的学习路径页面。

### 学习路径

学习路径页面展示了完整的课程体系，按 Track > Module > Lesson 三级结构组织。

**使用方法**:

1. **左侧面板**: 选择技术栈（Track），展开查看模块（Module）和课程（Lesson）
2. **主面板**: 显示当前选中课程的 Markdown 内容，支持语法高亮
3. **课程标记**: 可标记课程为已完成（快捷键 `Ctrl+Enter`）
4. **笔记功能**: 在课程页面中记录学习笔记，自动保存
5. **阅读模式**: 支持沉浸式阅读对话框

**课程状态**:

- 未开始: 灰色标记
- 进行中: 蓝色标记
- 已完成: 绿色标记

### 练习中心

练习中心提供编程练习和自动评测功能。

**使用方法**:

1. **筛选练习**: 使用顶部下拉框按技术栈和难度筛选
2. **选择练习**: 从左侧列表中选择要做的练习
3. **阅读题目**: 主面板显示题目描述、提示和要求
4. **编写代码**: 在代码编辑器中编写解答（支持 Python/SQL/C/C# 语法高亮）
5. **运行代码**: 点击**运行**按钮查看代码输出（仅 Python）
6. **提交评测**: 点击**提交**按钮进行自动评测
7. **查看反馈**: 评测结果显示得分、通过状态和详细反馈

**评分规则**:

- 语法检查: 20 分
- 代码结构: 20 分
- 执行成功: 10 分
- 关键对象: 10 分
- 测试用例: 40 分
- 通过阈值: 70 分

**草稿保存**: 编辑器中的代码会自动保存为草稿，下次打开同一练习时自动恢复。

### 融合项目

融合项目页面展示综合性项目，帮助你将多个知识点整合到实际项目中。

### 算法动画

算法动画页面提供经典算法的可视化演示，帮助理解算法执行过程。

### AI 工作台

AI 工作台是与 AI 导师对话的界面，支持多种使用模式。

**两种显示模式**:

- **独立页面**: 通过左侧导航或顶部按钮进入，左侧会话列表 + 右侧聊天区域
- **侧边助手**: 通过 Dock 面板在任何页面右侧显示，适合边学边问

**会话管理**:

- **新建会话**: 点击"新建"按钮，为不同主题创建独立会话
- **切换会话**: 在会话列表中点击切换
- **重命名会话**: 选中会话后点击"重命名"
- **删除会话**: 选中会话后点击"删除"（至少保留一个会话）
- **设为当前**: 将选中会话设为默认会话

**快捷提问**:

页面底部提供三个快捷提问按钮：

- **解释当前课程**: 请 AI 解释当前打开课程的核心知识点
- **分析当前代码**: 请 AI 分析当前编辑器中的代码或报错
- **拆解当前项目**: 请 AI 将当前项目拆解为可执行步骤

**知识库**:

AI 助手的回答会结合三层知识库：

- **基础知识库**: 内置课程体系摘要
- **个性知识库**: 根据你的学习进度和练习记录生成
- **扩展知识库**: 你手动添加的文件摘录

## 配置指南

### AI 模型配置

DevLearnerAI 支持任何 OpenAI 兼容 API 端点。

**配置步骤**:

1. 打开 AI 工作台或侧边助手
2. 点击"AI 设置"
3. 在"模型连接"标签页中填写：
   - **API Host**: API 端点地址（必须以 `https://` 开头）
   - **API Key**: API 密钥
4. 点击"获取模型"获取可用模型列表
5. 选择模型后点击"保存配置"

**支持的 API 端点**:

- OpenAI 官方 API: `https://api.openai.com/v1`
- 其他 OpenAI 兼容端点（如 Azure OpenAI、本地部署的兼容服务等）

**API 密钥安全存储**:

- Windows: 存储在 Windows Credential Manager 中（加密）
- 其他平台: 使用 keyring 库或 Base64 编码文件

### 知识库配置

在 AI 设置的"知识库"标签页中管理知识库：

- **基础知识库**: 自动从课程内容生成，无法手动编辑
- **个性知识库**: 根据学习数据自动生成，包含学习进度和练习记录
- **扩展知识库**: 点击"添加文件"选择本地文件加入知识库

通过勾选/取消勾选复选框控制各层知识库的启用状态。

### 数据存储位置

| 数据 | 位置 |
|------|------|
| 数据库 | `%APPDATA%/DevLearnerAI/data/app.db` |
| 日志 | `%APPDATA%/DevLearnerAI/logs/` |
| 缓存 | `%APPDATA%/DevLearnerAI/cache/` |
| 导出 | `%APPDATA%/DevLearnerAI/exports/` |
| 草稿 | `%APPDATA%/DevLearnerAI/drafts/` |
| API 密钥 | Windows Credential Manager (DevLearnerAI:mentor_api_key) |

> 在非 Windows 系统上，`%APPDATA%` 对应 `~/.devlearnerai/`

## 故障排除

### 应用无法启动

1. 确认 Python 版本: `python --version`（需要 3.9+）
2. 确认依赖已安装: `pip install -r requirements.txt`
3. 查看错误输出: 使用 `python dev_main.py` 启动以获取详细日志

### AI 助手无法连接

1. 确认 API Host 以 `https://` 开头（不支持 HTTP）
2. 确认 API Key 正确且未过期
3. 使用"测试连接"按钮验证配置
4. 检查网络连接和防火墙设置
5. 确认所选模型存在于 API 端点

### AI 回复异常

1. 检查 API 端点是否正常工作
2. 尝试切换到其他模型
3. 检查 API Key 的配额是否用尽
4. 创建新会话重试

### 代码执行失败

1. 确认代码中没有使用 `import` 语句（沙箱限制）
2. 确认没有访问双下划线属性（如 `__class__`）
3. 确认代码没有死循环（默认 3 秒超时）
4. 确认标准输出不超过 12KB

### 练习评测不通过

1. 仔细阅读评测反馈中的具体说明
2. 检查是否缺少必要的关键字或结构
3. 对于 SQL 练习，确认结果集与预期一致
4. 对于 Python 练习，确认所有测试用例通过

### 数据丢失

1. 学习数据存储在 `%APPDATA%/DevLearnerAI/data/app.db`
2. 如果数据库损坏，可尝试从备份恢复
3. 删除 `app.db` 文件后重启应用会创建新数据库（历史数据将丢失）
4. 定期备份 `app.db` 文件以防止数据丢失

### 界面显示异常

1. 确认 PyQt5 版本 >= 5.15
2. 检查系统 DPI 设置
3. 尝试调整窗口大小
4. 重启应用
