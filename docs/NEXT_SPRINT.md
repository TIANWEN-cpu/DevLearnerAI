# CodeHelper -- 下一轮冲刺计划

> 制定日期：2026-06-02
> 基于：HIGH_ROI_IMPROVEMENTS.md、SPRINT_REVIEW.md、当前任务清单
> 当前成熟度：L2 MVP | 目标：L2+ (用户体验和 AI 能力维度升级)
> 冲刺周期建议：2 周（10 个工作日）

---

## 一、冲刺背景

上一轮冲刺（Sprint 4-14）完成了 21 项功能特性、200+ 测试用例、12 项性能优化、30+ 文档。但 Sprint 11+ 后续迭代因 API 429 限流导致完成率仅 40%，多个关键任务中断。

当前最大短板是**用户体验（L1）**和**AI 能力（L1）**两个维度。新用户首次打开看到空白列表无任何引导；AI 对话不知道用户在做什么题、写了什么代码；统计数据仍在 localStorage 中，清缓存即丢失。本轮冲刺聚焦于消灭这些 L1 维度 + 补齐中断的测试覆盖。

---

## 二、冲刺目标（5 项，按优先级排序）

### 目标 1：统计数据迁移至 SQLite（P0 数据安全）

| 维度 | 说明 |
|------|------|
| **优先级** | P0 -- 消除数据丢失风险 |
| **预计工时** | 3 天 |
| **当前状态** | in_progress（迁移方案设计完成，实现中断） |

**具体任务**

1. 审计 `StatsView.tsx` 和 `appStore.ts` 中所有 localStorage 读写点，逐一列出
2. 基于已有 `submissions`、`mistakes`、`chat_history` 表构建统计 SQL 查询：
   - 每日/每周做题数量和通过率
   - 按难度分布的题目完成情况
   - AI 对话频次和时长
   - 错题复习次数和转化率
3. 在 `electron/ipc/database.ts` 新增统计查询 IPC handler
4. 利用 `analytics_events` 表记录关键行为事件
5. 添加 JSON 格式数据导出功能
6. 清理所有 localStorage 业务数据读写，仅保留 UI 偏好

**成功标准**

- localStorage 中不再存储任何业务数据
- 清除浏览器缓存后统计数据完整保留
- 统计面板所有图表和数字来自 SQLite
- 支持 JSON 格式一键导出

**风险与缓解**

| 风险 | 概率 | 缓解 |
|------|------|------|
| 历史 localStorage 数据迁移丢失 | 高 | 首次启动检测并自动迁移旧数据 |
| 统计 SQL 查询性能不足 | 中 | 添加索引 + 结果缓存 5 分钟 |

### 目标 2：AI 上下文感知注入（P1 产品核心差异化）

| 维度 | 说明 |
|------|------|
| **优先级** | P1 -- AI 能力 L1->L2 的关键 |
| **预计工时** | 3 天 |
| **当前状态** | pending |

**具体任务**

1. 在 `src/stores/chatStore.ts` 的 `sendMessage` 方法中，读取 `problemStore` 和 `editorStore` 当前状态
2. 构建 context system message，注入：当前题目标题/描述/难度/标签、编辑器中的代码、最近提交结果
3. 在 `electron/ipc/ai.ts` 的 `injectMemories` 基础上扩展，将 context message 插入 messages 数组头部
4. 在设置中添加开关：用户可关闭自动上下文注入
5. 在 `ChatView.tsx` 中显示当前注入的上下文状态（如"正在讨论：两数之和 (Easy)"）
6. 在 `AISidebar.tsx`（题目侧边栏）复用同一上下文注入逻辑

**成功标准**

- AI 回答中提及当前题目名称或代码变量名的比例 > 80%
- 用户手动描述题目的消息数量下降 > 50%
- 设置中可控制上下文注入开关

**风险与缓解**

| 风险 | 概率 | 缓解 |
|------|------|------|
| 上下文注入增加 token 消耗 | 中 | 限制注入长度（题目描述 500 字、代码 2000 字截断） |
| 编辑器无代码时注入空内容 | 低 | 仅在有实际内容时注入对应字段 |

### 目标 3：新用户引导流程完善（P1 最高 ROI）

| 维度 | 说明 |
|------|------|
| **优先级** | P1 -- 用户体验 L1->L2 的关键 |
| **预计工时** | 2 天 |
| **当前状态** | in_progress（onboarding store 已创建，UI 未集成主流程） |

**具体任务**

1. 创建 `src/components/OnboardingWizard.tsx`，实现 3 步引导：
   - **欢迎页**：产品定位说明 + 核心功能文字预览
   - **API Key 配置**：引导添加 AI 配置，提供"智能粘贴"解析 OpenAI 格式 URL

2. 在 `App.tsx` 中检查 `settings` 表的 `onboarding_completed` 键，条件渲染
3. 在设置页面添加"重新运行入门引导"入口

**成功标准**

- 新用户 2 分钟内完成引导进入第一道题
- 引导完成后自动进入主界面
- 已完成用户不再看到引导页
- 设置中可重新触发引导

**风险与缓解**

| 风险 | 概率 | 缓解 |
|------|------|------|
| 引导流程过长 | 低 | 3 步精简 + "跳过" 按钮 |
| 首题选择不当 | 低 | 选择经典 easy 题（如两数之和），有完整题解 |

### 目标 5：生产代码清理 + 结构化日志（P1 工程质量）

| 维度 | 说明 |
