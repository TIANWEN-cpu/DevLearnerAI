# DevLearner AI 无障碍功能文档

本文档介绍 DevLearner AI 桌面编程学习平台的无障碍功能、使用方法和已知限制。

---

## 1. 无障碍功能概述

DevLearner AI 基于 PyQt5 构建，在设计时融入了以下无障碍支持：

### 键盘导航

- 所有交互控件（按钮、输入框、下拉列表、选项卡、列表等）均可通过 Tab 键聚焦。
- 聚焦时会显示高亮轮廓（2px 实线蓝色边框 + 2px 偏移），便于视觉识别当前焦点位置（见 `app/styles.py` 中的 `focus-visible` 规则）。
- 弹出对话框和弹窗均支持 Escape 键关闭。
- 侧边栏导航按钮、页面切换、练习操作等均提供快捷键，无需鼠标即可完成全部核心操作。

### 屏幕阅读器支持

- 主要界面元素均设置了 `setAccessibleName` 和 `setAccessibleDescription`，包括：
  - 侧边栏导航按钮、页面标题和副标题
  - 学习进度条、目标进度条、图表控件
  - 算法可视化控件（算法选择、速度调节、数据数量、运行/暂停/停止按钮）
  - AI 对话面板的快捷操作按钮
  - 练习界面的代码编辑器和提交按钮
  - 收藏、成就、数据分析等弹窗
- 状态栏的连接状态指示器提供 `accessibleName` 和 `accessibleDescription`。
- 切换主题、调整字体大小等操作会通过状态栏的 `setAccessibleDescription` 发布通知。

### 主题与对比度

- 提供浅色和深色两套主题，用户可通过 `Ctrl+T` 即时切换。
- 浅色主题的文字颜色（`#0f1a2e`）与背景（`#eef4f8`）之间的对比度满足 WCAG AA 标准。
- 深色主题的文字颜色（`#f1f5f9`）与背景（`#0c1222`）之间的对比度同样满足 WCAG AA 标准。
- 所有交互控件的聚焦状态使用主题强调色（浅色 `#2563eb`，深色 `#60a5fa`）作为轮廓色，确保可见性。

### 字体大小可调

- 提供三档字体大小预设：小（small）、中（medium，默认）、大（large）。
- 可通过侧边栏底部的字体大小按钮实时切换，无需重启应用。
- 大字号模式对视力不佳的用户尤为友好。

### Tooltip 提示

- 所有按钮和关键控件均设置了 Tooltip（鼠标悬停时显示功能说明）。
- 图表控件在鼠标悬停数据点时显示具体数值。
- 快捷键提示集成在 Tooltip 中（如导航按钮显示"学习路径 (Ctrl+2)"）。

### 代码语法高亮

编辑器中的语法高亮使用以下配色，均基于深色背景 `#1a2740` 校准：

| 语法元素 | 颜色 | 说明 |
|:---|:---|:---|
| 关键字 | `#2563eb` | 蓝色加粗 |
| 内置函数 | `#a16207` | 琥珀色 |
| 类型名 | `#7c3aed` | 紫色 |
| 数字 | `#0f9d58` | 绿色 |
| 字符串 | `#b45309` | 深橙色 |
| 注释 | `#6b7280` | 灰色斜体 |

## 4. 字体大小配置

应用内置三档字体大小预设，可通过侧边栏底部的 A- / A / A+ 按钮切换：

| 档位 | 基准字号 | 标题字号 | 副标题字号 | 代码字号 |
|:---|:---|:---|:---|:---|
| 小（small） | 28px | 54px | 36px | 26px |
| 中（medium，默认） | 33px | 63px | 41px | 31px |
| 大（large） | 40px | 72px | 48px | 38px |

**配置文件入口**：`app/styles.py` 中的 `FONT_SIZES` 字典。

**切换方式**：

1. 在应用界面左侧边栏底部，找到字体大小按钮（A- / A / A+）。
2. 点击即可即时生效，无需重启。
3. 状态栏会提示"字体大小已调整为：大/中/小"。

**默认字体族**：

- 界面字体：`Microsoft YaHei UI`（Windows），回退至系统无衬线字体。
- 等宽字体：`Consolas`（代码编辑区域）。

## 5. 已知限制

### 当前版本不支持的功能

1. **高对比度模式检测**：应用未检测操作系统的高对比度模式设置。使用系统级高对比度模式的用户可能需要手动切换到深色主题以获得更好的体验。

2. **屏幕阅读器兼容性**：`setAccessibleName` / `setAccessibleDescription` 已覆盖大部分控件，但以下区域的无障碍标签可能不够完整：
   - 自定义绘制的图表控件（折线图、柱状图、雷达图、热力图）仅提供 `AccessibleDescription` 文本摘要，无法逐数据点交互。
   - 代码编辑区域（QPlainTextEdit）内的语法高亮对屏幕阅读器不可见。
   - Markdown 渲染后的课程内容区域的结构化导航有限。

3. **键盘快捷键冲突**：在 AI 对话输入框等文本编辑区域中，`Ctrl+N` 等快捷键可能与系统或输入法快捷键冲突。此时快捷键可能被输入框拦截而无法触发全局操作。

4. **macOS / Linux 平台差异**：
   - 快捷键目前统一使用 `Ctrl` 前缀。macOS 用户可能期望使用 `Cmd` 键（此适配尚未完成，参见项目待办事项 #285）。
   - 字体回退链主要针对 Windows 优化。macOS 和 Linux 上的字体显示可能存在差异。

5. **动画与运动**：算法可视化、加载动画、渐入效果等使用了 CSS 动画和 QTimer 驱动的帧动画。目前未提供全局"减少动画"开关，对动画敏感的用户可能需要避免进入算法可视化页面。

6. **国际化与无障碍标签**：无障碍标签（`AccessibleName` / `AccessibleDescription`）目前以中文硬编码或通过 i18n 系统动态生成。切换到英文界面时，部分自定义控件的无障碍描述可能仍为中文。

7. **阅读器放大功能**：课程阅读器（reader_dialog）提供了独立的放大/缩小字体按钮，但放大范围和持久化设置有限。

8. **触屏支持**：应用为桌面鼠标/键盘交互设计，未针对触屏设备优化点击区域大小。

### 计划改进

- macOS 键盘快捷键适配（Cmd 替代 Ctrl）
- "减少动画"全局开关
- 更完整的屏幕阅读器图表数据支持
- 系统高对比度模式自动检测
