# 配置与选项：把“写死在代码里”的东西拿出来

## 学习目标

- 理解配置为什么重要
- 知道配置值和业务逻辑不该混在一起
- 会读懂最小配置读取思路

## 先修知识

- 类
- 文件或 JSON 更容易串起来

## 为什么这节课重要

如果你的 API 地址、超时时间、文件路径都写死在代码里，后面改起来会很痛。  
配置存在的价值就是把“会变化的东西”收出去。

## 最小直觉

- 业务逻辑：程序做什么
- 配置：程序怎么做、去哪里做

## 典型配置项

- API Host
- 超时时间
- 日志级别
- 数据库连接字符串

## 常见错误

- 配置和业务常量混在一起
- 配置名取得太模糊
- 出错时不告诉用户具体缺了哪个配置

## 小练习

- 设计一个 `ApiHost` 和 `TimeoutSeconds` 配置模型
- 思考哪些值该做配置，哪些值更适合留在代码里

## 课后总结

- 配置是在帮你管理变化
- 越早把变化收口，项目越容易维护
