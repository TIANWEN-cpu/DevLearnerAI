# 函数、参数与模块拆分

## 学习目标
- 理解函数是“封装一段可复用逻辑”
- 区分参数、返回值、局部变量
- 开始学会拆分模块

## 为什么要写函数
如果一段逻辑会重复出现，或者太长难读，就应该考虑函数：

```python
def greet(name):
    return f"你好，{name}"
```

调用时：

```python
message = greet("Alice")
print(message)
```

## 参数和返回值
- 参数：函数接收的输入
- 返回值：函数执行后的结果

很多初学者会把 `print()` 和 `return` 混掉：

```python
def add(a, b):
    print(a + b)   # 只是打印
```

更通用的写法是：

```python
def add(a, b):
    return a + b
```

## 局部变量
函数内部定义的变量，默认只在函数内部可用：

```python
def area(width, height):
    result = width * height
    return result
```

`result` 是局部变量，函数外部通常不直接访问它。

## 把脚本拆成模块
当文件变长时，不要所有内容都塞一个 `.py` 里。比如：

- `main.py`：程序入口
- `utils.py`：通用函数
- `models.py`：数据模型

这种拆分会直接影响后面做项目时的清晰度。

## 设计函数时问自己三个问题
1. 这个函数的输入是什么
2. 它的输出应该是什么
3. 它是否做了太多事

如果一个函数既读文件、又改数据、又打印结果，通常说明还可以继续拆。

## 本课总结
函数是从“会写语句”到“会组织代码”的关键跳板。后面写数据库操作、项目逻辑和接口处理时，函数设计能力会直接决定代码质量。
