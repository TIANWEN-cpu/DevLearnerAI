# 面向对象与数据建模

## 学习目标
- 理解类和对象分别代表什么
- 会把“数据 + 行为”组织到一起
- 知道什么时候该用类，什么时候不用

## 类不是高级装饰品
类最核心的用途，是把同一类对象的属性和行为组织起来。

例如“学生”可以表示为：

```python
class Student:
    def __init__(self, name, score):
        self.name = name
        self.score = score

    def is_passed(self):
        return self.score >= 60
```

创建对象：

```python
tom = Student("Tom", 88)
print(tom.is_passed())
```

## `self` 是什么
`self` 指向当前实例本身。它不是关键字，但约定俗成必须这么写。

当你写：

```python
tom.is_passed()
```

本质上等价于：

```python
Student.is_passed(tom)
```

## 什么时候适合用类
- 你有一组相关数据需要一起管理
- 这些数据还带有行为
- 未来很可能会创建多个相似对象

不适合的情况：
- 只是一次性的小计算
- 一个简单函数就能说明白

## 建模比语法更重要
比如图书管理系统里，一本书有哪些属性？
- 标题
- 作者
- 是否借出
- 编号

这些属性决定了你的类设计，也会影响后续数据库表结构。

## 常见错误
- 类里只放属性，不放行为
- 把所有逻辑都堆进 `__init__`
- 还没必要用类时硬上类

## 本课总结
面向对象的本质是建模，不是炫技。你能不能把现实对象抽象成合适的数据和行为，远比会不会背继承语法更重要。
