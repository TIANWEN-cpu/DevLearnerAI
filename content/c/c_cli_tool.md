# 用 C 写一个命令行小工具

## 学习目标
- 把输入、处理、输出串成一个可运行的工具
- 学会控制功能范围，不一上来就做过头
- 开始形成“作品意识”

## 一个很合适的练手方向
- 文本行计数器
- 简单待办事项工具
- 关键字搜索器

这类程序的好处是：  
小、完整、能跑、能展示。

## 做项目时的顺序
1. 先定义最小功能
2. 再把数据结构定下来
3. 最后才补体验优化
