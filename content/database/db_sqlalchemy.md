# SQLAlchemy 与 ORM 入门

## 学习目标
- 理解 ORM 解决什么问题
- 知道模型、引擎、会话的大致角色
- 能判断什么时候适合 ORM，什么时候直接写 SQL

## ORM 是什么
ORM 的核心思想是：把数据表映射成类，把记录映射成对象。

例如你可以定义：

```python
class Book(Base):
    __tablename__ = "books"
```

然后通过对象方式新增、查询、更新。

## ORM 的价值
- 减少重复 SQL 拼接
- 更容易和 Python 代码组织结合
- 复杂项目里更利于维护模型关系

## ORM 的代价
- 抽象更多，初学者容易“只会调用不会理解”
- 性能问题更容易被隐藏
- 复杂查询最后还是要理解 SQL

## 你必须记住的一点
学习 ORM 不是为了跳过 SQL，而是在已经懂 SQL 的前提下，多一种表达方式。

## 建议掌握的最小链路
1. 创建 `engine`
2. 定义模型类
3. 创建 `Session`
4. 新增一条记录
5. 查询记录
6. 提交事务

## 本课总结
ORM 是工程工具，不是魔法。真正厉害的人，既能写 ORM，也知道它背后的 SQL 大概在做什么。
